# GST Return Automation Suite

HTML-based frontend prototype for Amazon GST and Tally automation.

## What it does

- Upload Amazon `MTR B2B`, `MTR B2C`, and `STR` reports separately or together
- Parse CSV or Excel files in the browser
- Detect source columns and show report-specific mapping review
- Build GST summary buckets for B2B, B2C, and stock transfer activity
- Generate working outputs for `GSTR-1 B2B`, `GSTR-1 B2CS`, `HSN`, and `GSTR-3B`
- Generate draft ledgers and vouchers for Tally import
- Export import-ready `Tally XML`, `ledger CSV`, `voucher CSV`, and `GST summary JSON`
- Optionally save processed run history to Supabase

## Local Run

Open `index.html` in a browser, or deploy as a static site.

## Supabase Table Suggestion

Create a table named `gst_runs` with columns:

- `id` bigint generated by default as identity primary key
- `company_name` text
- `uploaded_reports` jsonb
- `summary_json` jsonb
- `vouchers_json` jsonb
- `seller_summary_json` jsonb
- `gstr1_json` jsonb
- `gstr3b_json` jsonb
- `created_at` timestamptz default now()

Suggested SQL:

```sql
create table if not exists gst_runs (
  id bigint generated by default as identity primary key,
  company_name text,
  uploaded_reports jsonb,
  summary_json jsonb,
  vouchers_json jsonb,
  seller_summary_json jsonb,
  gstr1_json jsonb,
  gstr3b_json jsonb,
  created_at timestamptz default now()
);
```

## Environment Variables

Set these on Vercel:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `SUPABASE_TABLE`

Use `.env.example` as template for local reference.

## Vercel Deploy

1. Push these files to a Git repository.
2. Import the repository into Vercel.
3. Add the environment variables from `.env.example` in Vercel Project Settings.
4. Deploy. The frontend stays static and `/api/save-run` runs as a Vercel serverless function.

## Important Note

This is now a stronger working prototype with GST outputs, but final exact production behavior still depends on your real Amazon sample files. Before real production use, we should still lock:

- strict column mapping presets for your exact Amazon report format
- final bucket rules after validating your real `MTR B2B`, `MTR B2C`, and `STR` columns
- validation and error reporting
- exact Tally XML structure based on your company voucher settings
- authentication and secure Supabase API handling
